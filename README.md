<h2>Rules for contributing<sup>1</sup></h2>
<ul>
  <li>
    Formatting:
    <ul>
      <li>Generally we use CamelCase with a Caps starting letter for functions</li>
      <li>Constants both local and symbolic i.e. C-code and Preprocessor are to be written in all caps</li>
      <li>Any names are to be clearly descriptive and distinct, however they must not be overly long</li>
    </ul>
  </li>
  <li>
    Gen-AI:
    <ul>
      <li>Generally there is ***no*** Gen-AI use.</li>
      <li>In general in cases where Gen-AI may be used, it is advised to ask/use the project group first.</li>
      <li>if Gen-AI is used e.g. for debugging it is to be clearly stated.</li>
    </ul>
  </li>
  <li>
    Sources:
    <ul>
      <li>When external sources are used, they are to be clearly stated and linked.</li>
      <li>This especially applies to ***copied code***, which generally is to be kept to a minimum.</li>
      <li>Sources should additionally be made available in our Discord Server</li>
    </ul>
  </li>
</ul>
<hr>
<sup>1</sup> Failing to adhere to these rules can result in being banned from contributing to the codebase
